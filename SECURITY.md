# Security Policy

## Supported Versions

These versions are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.8.x   | :white_check_mark: |
| < 0.8   | :x:                |

## Vulnerabilities discovered with AI assistance

Issues that are discovered primarily by the use of publicly-available automated
AI tools should skip responsible disclosure. Just file a bug for these instead.
The purpose of responsible disclosure is to give maintainers a chance to fix
vulnerabilities before they are exploitable. These issues are effectively
already public, so disclosure doesn't add any value. Adding them to the issue
tracker instead offers the opportunity for contributors to deduplicate and fix
them. See this [LKML thread] for the Linux kernel project's related resoning.

[LKML thread]: https://lkml.org/lkml/2026/5/6/1391

## Reporting a Vulnerability

Please report vulnerabilities by emailing djkoloski (at) gmail (dot) com. I will
respond within 24 hours and update you at least once a week on progress towards
fixing the issue.
